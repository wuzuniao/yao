## 2026-06-29 - 修复注册流程双重故障（网络错误 + 密码哈希失败）

### 问题现象
1. Phase 1：注册页点击"获取验证码"提示"网络请求失败:fail"（已勾选"不校验合法域名"）
2. Phase 2：网络修复后，验证码能收到，但提交注册提示"验证码错误或已过期"（1分钟内操作，未超时）

### 根因分析（科学调试流程 + 插桩取证）

#### Phase 1: 网络请求失败
1. `netstat -ano | findstr :8000` → 无输出，8000 端口无监听进程
2. `python -c "import pydantic_settings"` → ModuleNotFoundError，后端依赖未安装
3. `pip install -r requirements.txt` → 失败，`python-jose` 在 Python 3.14 上无可用版本
4. 根因链：python-jose 不可用 → pip install 整体失败 → 依赖缺失 → 后端无法启动 → 端口无监听 → 连接拒绝 → request:fail

#### Phase 2: 密码哈希失败（通过 verify_code 插桩日志定位）
1. 插桩日志显示第一次提交 `verify_code 验证通过`，但随后 `(trapped) error reading bcrypt version`
2. `get_password_hash('Test1234!')` 直接测试 → `ValueError: password cannot be longer than 72 bytes`
3. 根因：passlib 1.7.4 访问 `bcrypt.__about__.__version__`（bcrypt 5.0.0 已移除该属性）→ passlib 内部 `detect_wrap_bug` 调用 `bcrypt.hashpw` 抛异常
4. 连锁反应：哈希失败 → register 抛 ValueError → 400 → 验证码被 verify_code 消费（一次性）→ 第二三次提交"记录不存在"

### 修复方案

#### Phase 1: 网络请求失败
1. **安装缺失依赖**：pip install pydantic-settings passlib[bcrypt] python-multipart email-validator（清华镜像源）
2. **替换 python-jose 为 pyjwt**：`requirements.txt` 中 `python-jose[cryptography]>=3.3.0` → `pyjwt>=2.8.0`
   - 原因：python-jose 已过时且不支持 Python 3.14；当前代码未使用 JWT；pyjwt 是现代替代方案
3. **启动后端服务**：uvicorn app.main:app --host 0.0.0.0 --port 8000

#### Phase 2: 密码哈希失败
1. **移除 passlib，直接使用 bcrypt 原生 API**（`app/core/security.py`）：
   - `passlib.CryptContext` → `bcrypt.gensalt()` + `bcrypt.hashpw()` + `bcrypt.checkpw()`
   - 原因：passlib 1.7.4 自 2020 年停止维护，与 bcrypt 5.0.0 不兼容；bcrypt 5.0.0 原生 API 更简洁
2. **更新 requirements.txt**：`passlib[bcrypt]>=1.7.4` → `bcrypt>=4.0.0`

### 验证结果
- Phase 1: send-code 端点 200 `{"code":0,"msg":"验证码已发送"}`，邮件成功发送
- Phase 2: 端到端服务层测试通过：
  - 验证码生成 → verify_code 验证通过 → INSERT users → 注册成功（id=1）→ 密码校验 True
- 完整注册流程无错误，密码哈希正常工作

### 影响范围：
- 后端：requirements.txt（python-jose→pyjwt, passlib→bcrypt）、app/core/security.py（重写为 bcrypt 原生 API）
- 前端：无修改（前端代码无缺陷，根因为后端依赖问题）
- 调试记录：debug-sendcode-network-error.md（已标记 [FIXED]，含完整插桩证据）

## 2026-06-29 - 注册页失焦验证 + 获取验证码网络错误修复

### 主要变动：

1. **表单失焦验证逻辑优化（register.vue）**
   - 新增 `errors` reactive 对象，按字段存储错误信息
   - 新增 `validateField(field)` 函数：失焦时按字段触发对应校验策略并写入 errors
   - 新增 `handleBlur(field)` 包装函数：先恢复 placeholder 原始色，再触发字段校验；密码字段失焦时联动重新校验确认密码
   - 新增 `handleFocus(field)` 函数：聚焦时清空该字段错误（输入清错）
   - 新增 `validateConfirmPassword` 函数：校验非空 + 与密码一致性
   - `handleRegister` 最终校验改为调用 `validateField` 写入 errors，同步展示视觉反馈
   - 所有输入框 `@blur` 改为 `handleBlur('字段名')`，`@focus` 改为 `handleFocus('字段名')`

2. **错误视觉反馈（register.vue 模板 + 样式）**
   - 输入框错误态：`:class="{ 'register-page__input--error': errors.字段 }"` 绑定红色边框
   - 验证码输入框错误态：`register-page__code-input--error` 红色边框
   - 每个字段下方新增 `<text v-if="errors.字段">` 错误文字提示（红色 #e5484d，12px）
   - 新增 CSS：`.register-page__input--error` / `.register-page__code-input--error`（box-shadow 红色边框）、`.register-page__error-text`（红色错误文字）

3. **获取验证码网络错误修复（request.js）**
   - 根因：微信小程序默认校验域名白名单，请求 `http://localhost:8000` 被拦截触发 fail 回调，原代码丢弃 errMsg 诊断信息
   - 修复 `fail` 回调：保留 `err.errMsg` 诊断信息，针对常见错误给出明确指引：
     - 域名未配置 → 提示在微信开发者工具勾选「不校验合法域名」
     - 连接拒绝 → 提示确认后端已启动（localhost:8000）
     - 超时 → 提示检查后端服务
   - Error 对象新增 `isNetworkError` 标识，区分网络错误与业务错误

4. **获取验证码重试机制（register.vue）**
   - 抽取 `requestCode()` 函数，`handleGetCode` 校验邮箱后调用
   - 网络错误（`isNetworkError`）：弹出 modal 提示原因 + 「重试/取消」按钮，确认则重新调用
   - 业务错误（如邮箱已注册）：toast 提示，不触发重试

### 影响范围：
- 前端：register.vue（脚本 + 模板 + 样式）、api/request.js（fail 回调错误处理）
- 构建验证：`npm run build:mp-weixin` 通过（exit 0，仅预存 Sass 警告）；register.vue 诊断无错误
- 使用提示：开发期需在微信开发者工具 → 详情 → 本地设置，勾选「不校验合法域名、web-view、TLS 版本以及 HTTPS 证书」，否则请求 localhost 会报网络错误
